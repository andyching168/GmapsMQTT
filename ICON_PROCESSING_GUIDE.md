# 圖標處理功能說明

## 概述
此功能會自動處理 Google Maps 導航通知中的圖標，並將其壓縮編碼後放入 JSON 數據中。整個處理流程經過精心設計，能將約 156x156 像素的彩色圖標壓縮成非常緊湊的字串。

## 處理流程

### 第一步：降維與預處理

#### 1.1 縮小尺寸 (Downscale)
- **輸入**：約 156×156 像素的彩色 Bitmap
- **處理**：使用雙線性插值將圖像縮放至 48×48 像素
- **輸出**：48×48 像素的彩色 Bitmap
- **目的**：大幅減少數據量（從 24,336 像素降至 2,304 像素）

#### 1.2 轉換灰階 (Grayscale)
- **輸入**：48×48 像素的彩色 Bitmap
- **處理**：使用標準灰階公式 `0.299R + 0.587G + 0.114B`
- **輸出**：48×48 像素的灰階 Bitmap
- **目的**：將 RGB 三通道簡化為單一亮度通道

### 第二步：圖像分析與分類

#### 2.1 計算低閾值 (T_low)
- 遍歷所有 2,304 個像素
- 計算整張圖的平均灰度值
- 此值作為「背景」和「前景」的分界線

#### 2.2 計算高閾值 (T_high)
- 遍歷所有灰度值高於 T_low 的像素
- 計算這些前景像素的平均灰度值
- 此值作為「灰色」和「白色」的分界線

#### 2.3 像素三態分類
- 建立 `int[2304]` 陣列存放分類結果
- 分類規則：
  - `灰度 < T_low` → `0` (背景)
  - `T_low ≤ 灰度 < T_high` → `1` (灰色)
  - `灰度 ≥ T_high` → `2` (白色)

### 第三步：數據壓縮與封裝

#### 3.1 行程長度編碼 (RLE)
- **輸入**：`int[1024]` 分類陣列
- **處理**：將連續相同的值壓縮成「值-次數」對
- **範例**：`[0,0,0,0,1,1,2,2,2]` → `[(0,4), (1,2), (2,3)]`
- **輸出**：緊湊的 `byte[]` 陣列
- **格式**：`[value1, count1, value2, count2, ...]`

#### 3.2 十六進位編碼 (Hex Encoding)
- **輸入**：RLE 壓縮的 `byte[]`
- **處理**：將每個 byte 轉換為兩位十六進位字符
- **範例**：`[0x0F, 0x1A]` → `"0F1A"`
- **輸出**：可放入 JSON 的 Hex 字串

## 數據壓縮效果

| 階段 | 數據量 | 說明 |
|------|--------|------|
| 原始圖像 | ~73 KB | 156×156 ARGB_8888 |
| 縮小後 | ~9 KB | 48×48 ARGB_8888 |
| 灰階後 | ~2.3 KB | 48×48 單通道 |
| 分類後 | 2304 int | 48×48 三態值 |
| RLE 壓縮 | ~100-450 bytes | 視圖案複雜度 |
| Hex 字串 | ~200-900 chars | 最終輸出 |

**壓縮比**：約 80:1 至 350:1

## JSON 輸出格式

```json
{
    "turnDirection": "left",
    "direction": "忠孝西路一段",
    "turnDistance": "100 公尺",
    "iconData": "00640110011E021C030A040C050A..."
}
```

### 欄位說明
- `turnDirection`：轉向方向（如 left, right, straight）
- `direction`：道路名稱
- `turnDistance`：距離轉向點的距離
- `iconData`：處理後的圖標數據（Hex 編碼的 RLE 壓縮字串）

## 程式碼使用

### ImageProcessor 類別

```kotlin
// 處理圖標並獲得壓縮字串
val hexString = ImageProcessor.processImage(bitmap)

// 解碼（用於驗證）
val bytes = ImageProcessor.hexToBytes(hexString)
val classificationArray = ImageProcessor.runLengthDecode(bytes)
```

### NavigationViewModel 整合

當通知圖標更新時，系統會自動：
1. 調用 `setLastIconHash(hash, bitmap)`
2. 內部執行 `ImageProcessor.processImage(bitmap)`
3. 將結果存儲在 `lastProcessedIcon`
4. 在 `generateNavigationJson()` 時自動加入 JSON

## 性能考量

- **處理時間**：約 15-80 ms（在現代 Android 設備上）
- **記憶體使用**：峰值約 10 KB（臨時緩衝區）
- **執行緒**：在主執行緒執行（處理速度快，不會造成 ANR）

## 優勢

1. **適中壓縮比**：原始圖像約 73 KB，壓縮後約 200-900 字符
2. **保留更多細節**：48×48 比 32×32 保留更多圖像細節和特徵
3. **快速處理**：整個流程在 15-80 ms 內完成
4. **易於傳輸**：Hex 字串可直接放入 JSON，適合 MQTT 傳輸
5. **可逆性**：提供解碼函數，可還原分類數據進行驗證
6. **更好的識別度**：更高的解析度讓圖標更容易辨識和重建

## 潛在用途

1. **遠端顯示**：接收端可根據壓縮數據重建簡化的圖標
2. **數據分析**：分析圖標變化模式
3. **機器學習**：作為輕量級特徵向量
4. **備份記錄**：以極小空間記錄導航歷史的視覺資訊
